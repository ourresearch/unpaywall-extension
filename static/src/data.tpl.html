About the data

Unpaywall is powered by a database of 100 million scholarly papers. Of these, about 16 million have links to fulltext open access, making it the largest open database of legal Open Access (by summer of 2018, we'll have around 18 million). It's also the most accurate: all records are keyed by DOI, and precision (98%) and recall (75%) of OA detection lead the industry (see our PeerJ paper for more on these numbers). We harvest OA content from over 50,000 unique sources including open repositories, OA journals, Hybrid journals, and services like DOAJ and Crossref.

There are lots of ways you can use the Unpaywall dataset in your own projects, depending on what you're trying to do:

  • Librarians: can integrate Unpaywall into their SFX, 360 Link, or Primo link resolvers, so library users can read OA copies in cases where there's no subscription access. Over 1000 libraries worldwide are now using this now.
  • Institutional Repository managers can use Unpaywall data to find OA resources that faculty have posted online, without depositing in their IR. These can be automatically ingested, significantly increasing IR coverage without needing to convince faculty to deposit. Repositories of all sizes have used Unpaywall data in this way. SwePub (national repository of Sweden) added 75,000 new OA records, increasing number of OA records by over 30%, while the smaller the Carleton University IR added 1000 OA records, doubling their fulltext coverage. There are a few good ways to access our data for this use case: downloading the complete dataset, check lists of records by DOI, or taking advantage of Unpaywall's integration into the Dimensions and Web of Science.
  • OA researchers can use Unpaywall to answer research questions about the current and historical state of open access. There are several ways to access the data for research: you can use the open API, or the R API wrapper, check a list of DOIs, or download the complete dataset. A particularly easy way to get started with research is to use Dimensions or Web of Science, which both have Unpaywall data integrated into their databases. That's the approach used by this recent paper. You might also check out the the canonical reference paper for the dataset, which gives a good overview of the data as a whole.
  • Enterprise users should check out our SLA in order to download and use the Unpaywall dataset for commercial projects. Once you've done that, we can help you set up a mirror database that allows you to keep an always-current version of Unpaywall's data hosted on your own servers. You can also use the API for any use (commercial or otherwise) for free.
